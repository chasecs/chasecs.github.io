JavaScript 数据类型判断

发表于 2017-10-14 | 更新于 2019-08-09 | 分类于 javascript
结合 lodash 源码，总结下判断 js 数据类型的方法，还有一点未解的困惑

JavaScript 的数据类型可以分为两大类： primitive 类型和 object 类型。

primitive 类型（原始类型）包括：

  • number
  • string
  • boolean
  • undefinded
  • null
  • Symbol（ ES6 新增）

除了 primitive，其余的都是 object（对象）

一、第一种方法 typeof

typeof 操作符，是判断 javascript 类型的基本方法，返回的结果是字符串:

typeof 能得到的结果如下：

Type Result
Undefined “undefined”
Null “object”
Boolean “boolean”
Number “number”
String “string”
Symbol (new in ECMAScript 2015) “symbol”
Host object (provided by the JS environment) Implementation-dependent
Function object (implements [[Call]] in ECMA-262 terms) “function”
Any other object “object”

四、== 和 === 的区别

判断数据类型时，经常要用到比较运算符（Comparison operators）== 和 ===，其中== 属于抽象比较（abstract comparison），=== 属于严格比较（strict comparison），两者的区别如下：

  • ==（Equality）符号两侧的操作数如果不是同一个类型，它会先把两侧的操作数转换成同一种类型，然后再进行严格比较（===）。
  • 如果==两侧的操作数是对象，则比较内部引用（internal references），如果他们都指向内存的同一个地方，则返回 true。
  • === 在运算时不会进行类型转换。
  • != 和 !== 的区别遵循上述原则

所以，当你要求两侧的操作数必须是同一种数据类型时，就应该使用 ===，否则可以用 ==。

进行数据转换时的规范

== 比较时，javascript 对字符串、数字、布尔值、对象的数据转换，遵循以下规范：

  • 比较数字和字符串，把字符串转成数字
  • 如果其中一个操作数时布尔值，布尔值 true 转成 1， false 转成 +0，
  • 对象和字符串或数字比较，javascript 会尝试用 valueOf 和 toString 把对象转成原始类型 字符串 或 数字，如果转换失败则报错
  • 只有当对象和原始类型的数据比较是，javascript 才尝试会把对象转成原始类型，如果是对象和对象之间的比较，则是判断它们是否指向同一个对象。
