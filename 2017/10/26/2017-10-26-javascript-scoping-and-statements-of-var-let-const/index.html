函数作用域

JavaScript 和其它很多编程语言有一个不同在于，它支持函数作用域（function-level scope），也就是说，它的变量的作用域会以函数为单位。其它编程语言比如 C，则支持块作用域（block-level scope）。

变量提升

提到 js 的作用域，就不得不提变量提升。变量提升可以理解为，变量和函数声明从它们在代码中出现的位置，“被移动”到作用域的最顶端。例如上文 Code_1 中的 bar() 为例，实际上会按以下形式处理：

之所以会有变量提升这种情况，是因为 JavaScript 引擎处理代码时是分两步走的，第一步是编译，工作内容就包括找到所有的声明，并用合适的作用域将它们关联起来（并在这个过程中可以优化性能），然后第二步才是执行代码（比如赋值）。

关于变量提升，有几点需要注意的：

  1. 只提升声明，不提升赋值

  2. 函数提升优先

避免变量提升带来混乱

虽然 js 有变量提升这种特性，但如果你的代码出现，变量调用早于声明的情况，多少会造成理解混乱。

所以最明智的做法是，总是在作用域的顶部先声明变量。比如，每个作用域只使用一个 var 关键字，而且在顶部使用，这样可以迫使把所有的变量都声明在顶部位置。

当然，进入 ES6 时代，是可以摈弃 var，拥抱 let 和 const。

使用 let 而不是 var

ES 6 语法引入了 let 语句。 let 和 var 最大区别是，let 所声明的变量支持块作用域。

source

此外，let 和 var 还有以下一些区别：

  • let 不存在“变量提升”

  • let 不能重复声明

使用 let 还需要注意，只要块级作用域内存在 let，它所声明的变量就“绑定”（binding）这个区域，形成一个暂时死区（temporal dead zone），不再受外部的影响。

相比 var ，使用 let 避免许多容易造成混乱的场景，所以在 能用 let 的场景就不要使用 var。

用 const 声明常量

const 和 let 一样，都是 ES6 引入的，它和 let 很相似，不过，用 const 声明的值不能被重新赋值，也不能重新声明。
